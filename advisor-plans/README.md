## Dependency notes

- **002 depends on 001** — DB-backed sessions are only safe once the HMAC
  signing secret is non-placeholder and fail-closed in production.
- **003 depends on 002** — binding the streaming token to the requester needs a
  reliable server-side session principal (the DB session id), which 002 introduces.
- **006 depends on 001** — the real Drive connector reads `GOOGLE_*` secrets;
  rotate + document them (001) before wiring code that consumes them.
- **004 is independent** — upload atomicity is a self-contained correctness fix
  on the service/action layer; it can land before or after the Drive work, but
  its real-world payoff arrives with 006.
- **008 first (no dependency)** — git init dává všem dalším plánům reálný SHA
  pro drift-check; ideálně před 009–011.
- **010 depends on 009** — náhledová proxy route musí použít read-only session
  variantu, kterou zavádí 009; jinak by 010 znovu zavlekl write-amplifikaci.
- **009/011 jsou nezávislé** — 009 je samostatná perf oprava session vrstvy,
  011 je admin UI nad existujícími službami; lze je dělat v libovolném pořadí.
- **012 depends on 010 + 011** — wizard používá náhledy (`posterUrl`, 010) a
  štítkovací akce/služby z 011 (čárka-split a našeptávač rozšiřují tentýž vstup).
  Stavět po slicech 1→4; každý slice je samostatně mergovatelný.
- **015/016/017 jsou nezávislé** (audit dávky `824f901`) — lze v libovolném
  pořadí; doporučené pořadí dle priority: 015 (P1 security) → 016 → 017.
